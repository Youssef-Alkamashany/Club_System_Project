### 🌟 Project Overview
The **Club Management System** is a technical showcase of core data manipulation strategies. It manages three primary entities—**Clubs, Members, and Sports**—while providing a professional **GUI** to interact with data and observe algorithmic performance in real-time.

---

### 🚀 Technical Core: Algorithms & Complexity
The application implements five distinct sorting algorithms and binary search, each with explicit **Big O** complexity tracking:

| Operation | Target Data | Algorithm | Complexity |
| :--- | :--- | :--- | :--- |
| **Sorting** | Clubs by Name | **Bubble Sort** | $O(n^2)$ |
| **Sorting** | Members by ID | **Selection Sort** | $O(n^2)$ |
| **Sorting** | Sports by Name | **Insertion Sort** | $O(n^2)$ |
| **Sorting** | Clubs by Name | **Merge Sort** | $O(n \log n)$ |
| **Sorting** | Members by ID | **Quick Sort** | $O(n \log n)$ (Avg) |
| **Searching** | Clubs/Members | **Binary Search** | $O(\log n)$ |

---

### 🛠️ System Architecture
The project follows an **Object-Oriented Programming (OOP)** approach with modular class structures:
* **Main Logic Engine:** Orchestrates data storage using `ArrayList` and manages the GUI event loop.
* **Entity Classes:** * `Club`: Manages branch data, managers, and locations.
    * `Member`: Handles unique IDs, contact info, and family details.
    * `Sport`: Tracks sport IDs and team distributions.
* **Dynamic GUI:** A custom-styled terminal-themed interface using `JFrame`, `GridLayout`, and custom color palettes for a professional developer experience.

---

### 📂 Key Features
* **Full CRUD Support:** Add and display records for clubs, members, and sports dynamically.
* **Real-time Performance Monitoring:** Output area displays the algorithm used and its time complexity after every operation.
* **Safe Input Handling:** Integrated error catching for numeric fields and invalid ID formats.
